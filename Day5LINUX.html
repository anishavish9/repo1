Assignments 1. write some 30 Linux commands.

  1. rm:

Description: Deletes the file.

Example: rm one.txt

  1. less:

Description: Displays the contents of a file one screen at a time.

Example: less fruits.txt

  1. grep:

Description: Searches for a pattern in a file.

Example: grep coconut fruits.txt

  1. find:

Description: Searches for files in a directory hierarchy.

Example: find Music

  1. gzip:

Description: Compresses a file (in zip).

Example: gzip three.txt

  1. gunzip:

Description: Decompresses a compressed file.

Example: gunzip three.txt

  1. bzip2:

Description: Compresses a file (higher compression ratio than gzip).

Example: bzip2 three.txt

  1. bunzip2:

Description: Decompresses a compressed file.

Example: bunzip2 three.txt.bz2

  1. du:

Description: Shows the disk usage of files and directories.

Example: du Music

  1. df:

Description: Shows the disk space usage of filesystems.

Example: df Music

  1. top:

Description: Displays the current system status.

Example: top

  1. ps:

Description: Lists the currently running processes.

Example: ps

  1. free:

Description: Shows the memory usage.

Example: free

  1. history:

Description: Shows the command history.

Example: history

  1. uniq:

Description: Removes duplicate lines from a file.

Example: uniq one.txt

  1. diff:

Description: Compares two files and shows the differences.

Example: diff one.txt two.txt

  1. curl:

Description: Transfers data from or to a server.

Example: curl -o myfile.txt http://example.com/myfile.txt

  1. clear:

Description: Clears the terminal.

Example: clear

  1. echo:

Description: Prints on terminal.

Example: echo "Hello, World!"

  1. man:

Description: Shows manual of any command.

Example: man ls

  1. export:

Description: Sets environment variables.

Example: export PATH=/new/path:$PATH

  1. crontab:

Description: Schedules tasks to run at regular intervals.

Example: crontab -e

  1. chmod:

Description: Changes file permissions.

u: User/owner permissions.

g: Group permissions.

o: Other permissions.

+: Add permissions.

–: Remove permissions.

=: Set permissions explicitly.

Example: chmod u+rwx file.txt

  1. chown:

Description: Changes file ownership.

Example: chown user file.txt

  1. chgrp:

Description: Changes group ownership.

Example: chgrp group file.txt

  1. umask:

Description: Sets default file permissions.

Example: umask 022

  1. tar:

Description: Creates or extracts archive files.

c: Create a new archive.

x: Extract files from an archive.

f: Specify the archive file name.

v: Verbose mode.

z: Compress the archive with gzip.

j: Compress the archive with bzip2.

Example: tar -czvf archive.tar.gz files/

  1. kill:

Description: Terminates a process.

Example: kill PID

  1. pkill:

Description: Terminates processes based on their name.

Example: pkill process_name

  1. uptime:

Description: Shows system uptime.

Example: uptime

  Assignment 3. explain absolute and relative path

Absolute Path

Specifies the complete location of a file or directory from the root.

Example: /home/anisha/documents/report.txt

Usage: Always gives the exact location, independent of the current directory.

Relative Path

Specifies the location of a file or directory relative to the current working directory.

Example: documents/report.txt (if current directory is /home/anisha/)

Usage: Shorter and more convenient for navigation within the same directory structure, depends on the current directory.

Key Difference: Absolute paths are fixed and start from the root, while relative paths are shorter and depend on the current directory.
